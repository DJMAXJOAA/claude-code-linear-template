# Coding Rules

> 이 파일은 프로젝트별로 커스터마이즈하여 사용한다. 아래는 스켈레톤 예시.

## §1 네이밍

| 대상 | 규칙 | 예시 |
|------|------|------|
| 변수/함수 | camelCase | `getUserName` |
| 클래스/타입 | PascalCase | `UserService` |
| 상수 | UPPER_SNAKE_CASE | `MAX_RETRY_COUNT` |
| 파일 | kebab-case | `user-service.ts` |

## §2 코드 품질

| 항목 | 지침 |
|------|------|
| 함수 길이 | 50줄 이내 권장 |
| 파일 길이 | 300줄 이내 권장 |
| 중첩 깊이 | 3단계 이내 (early return 활용) |
| 매직 넘버 | 상수로 추출 |
| 주석 | Why 위주. 자명한 코드에 불필요한 주석 금지 |

## §3 에러 처리

| 항목 | 지침 |
|------|------|
| 에러 전파 | 시스템 경계에서만 catch. 내부는 전파 |
| 에러 메시지 | 사용자 대면: 친절하게. 내부 로그: 상세하게 |
| 빈 catch 금지 | 최소 로깅 필수 |

## §4 테스트

| 항목 | 지침 |
|------|------|
| 테스트 네이밍 | `{행위}_when_{조건}_should_{결과}` 또는 서술형 |
| 핵심 로직 커버리지 | 80% 이상 권장 |
| 외부 의존성 | Mock/Stub 활용 |
