# /정리 — 사후 문서화

자유롭게 수정한 작업 내용을 정리하여 문서화한다.

## $ARGUMENTS

- **미지정**: git diff/staged/최근 커밋 자동 분석
- **{LINEAR-ID}**: 기존 이슈에 반영 (예: `/정리 PRJ-47`)
- **"{코멘트}"**: 작업 의도 힌트 (예: `/정리 "경로 리팩토링"`)
- **{LINEAR-ID} "{코멘트}"**: 조합 가능 (예: `/정리 PRJ-47 "경로 리팩토링"`)

---

## 실행

| 단계 | 행위 |
|------|------|
| 1 | `$ARGUMENTS` 파싱 — Linear ID / 코멘트 추출 |
| 2 | cleanup Skill 호출 |
| 3 | 결과 출력 |

> cleanup Skill 상세: [.claude/skills/cleanup/SKILL.md](../skills/cleanup/SKILL.md)
