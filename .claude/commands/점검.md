# /점검 — 수동 테스트 결과 분류 및 라우팅

수동 테스트 결과를 7유형으로 분류하고 적절한 처리 경로로 라우팅한다.

## $ARGUMENTS

테스트 결과 텍스트 (자유 형식). 미지정 시 사용자에게 입력 요청.

---

## 실행

triage Skill을 호출한다.

| 단계 | 행위 |
|------|------|
| 1 | `$ARGUMENTS`에서 테스트 결과 텍스트 파싱 |
| 2 | 활성 Issue의 Linear ID 확인 (있는 경우) |
| 3 | triage Skill 호출 — 7유형 분류 → G2 승인 → 라우팅 실행 |

> triage Skill 상세: [.claude/skills/triage/SKILL.md](../skills/triage/SKILL.md)
