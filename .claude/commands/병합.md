# /병합 — 브랜치 병합

지정된 브랜치를 병합한다.

## $ARGUMENTS

병합 방향 (예: `feature/auth -> main`). 미지정 시 현재 브랜치를 main에 병합.

---

## Phase 1: 병합 준비

| 단계 | 행위 |
|------|------|
| 1 | `$ARGUMENTS`에서 source → target 브랜치 파싱 |
| 2 | 현재 브랜치 상태 확인 (`git status`) |
| 3 | uncommitted 변경이 있으면 경고 + 중단 |
| 4 | source/target 브랜치 존재 확인 |

## Phase 2: 병합 실행

| 단계 | 행위 |
|------|------|
| 1 | target 브랜치로 checkout |
| 2 | `git merge {source}` 실행 |
| 3 | 충돌 발생 시: AI가 충돌 해결 제안 → `AskUserQuestion`으로 승인 |
| 4 | 병합 커밋 생성 |

## Phase 3: 병합 후 검증

| 단계 | 행위 |
|------|------|
| 1 | 빌드/테스트 실행 (있는 경우) |
| 2 | 병합 결과 출력 (커밋 수, 변경 파일) |
| 3 | source 브랜치 삭제 여부 사용자 확인 |
