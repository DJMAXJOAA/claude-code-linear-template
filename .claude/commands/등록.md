# /등록 — Issue 등록

새 Issue를 Linear에 등록하고 Git 문서 폴더를 생성한다. **단건** 및 **배치(일괄)** 등록을 모두 지원한다.

## $ARGUMENTS

자유 형식 텍스트. 입력 길이와 내용에 따라 자동으로 모드를 판별한다.

- **단건 모드**: type 힌트 + 짧은 제목/설명. 예: `feature 사용자 인증 기능`, `bug 로그인 실패`
- **배치 모드**: 긴 텍스트(spec 문서, 보고서, 기획서, 자유 프롬프트 등). AI가 이슈 후보를 추출한다
- **미지정**: `AskUserQuestion`으로 입력 요청

---

## 모드 판별

| 조건 | 모드 |
|------|------|
| 미지정 또는 `type + 한 줄 제목` 패턴 | **단건** |
| 여러 줄 텍스트, 문서 경로, 또는 복수 기능/버그 서술 | **배치** |

> 판별이 모호할 경우 `AskUserQuestion`으로 "단건 등록 / 배치 추출" 중 선택 요청

---

## 실행: 단건 모드

gen-hub Skill을 1회 호출한다.

| 단계 | 행위 |
|------|------|
| 1 | `$ARGUMENTS` 미지정 시 `AskUserQuestion`으로 type + 제목/설명 입력 요청 |
| 2 | `$ARGUMENTS`에서 type + 제목/설명 파싱 |
| 3 | gen-hub Skill 호출 — Linear Issue 생성 + `docs/issue/{LINEAR-ID}/_index.md` 생성 |
| 4 | 생성된 Issue 정보 출력 (LINEAR-ID, URL, type) |

---

## 실행: 배치 모드

입력 텍스트에서 이슈 후보를 추출하고, 사용자 확인 후 일괄 등록한다.

| 단계 | 행위 |
|------|------|
| 1 | **텍스트 분석**: 입력 텍스트(또는 파일 경로 지정 시 해당 파일 내용)를 분석하여 이슈 후보 추출 |
| 2 | **후보 목록 제시**: 테이블 형태로 사용자에게 제시. 각 항목에 AI 추천 type, 제목, 설명, priority 포함 |
| 3 | **사용자 확인**: `AskUserQuestion`으로 수정/삭제/추가 요청. 확인 시 다음 단계 진행 |
| 4 | **일괄 생성**: 확인된 각 항목에 대해 gen-hub Skill을 순차 호출 |
| 5 | **결과 요약**: 생성된 전체 Issue 목록 출력 (LINEAR-ID, URL, type) |

### 후보 목록 제시 형식

```
| # | type | 제목 | 설명 (요약) | priority |
|---|------|------|-------------|----------|
| 1 | feature | ... | ... | Medium |
| 2 | bug | ... | ... | High |
| 3 | improvement | ... | ... | Low |
```

> 사용자는 번호로 수정/삭제 지시 가능. 예: "2번 삭제", "3번 type을 feature로 변경", "추가: bug 로그인 오류"

### 제약

- 한 번에 최대 **10건**까지 등록. 초과 시 분할 안내
- gen-hub Skill 자체는 변경 없이 단건 호출 반복
