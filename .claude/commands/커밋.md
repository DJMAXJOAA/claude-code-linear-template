# /커밋 — Conventional Commits 자동 커밋

> **유틸리티 커맨드** — 4단계 게이트 면제.

현재 작업 디렉토리의 변경사항을 Conventional Commits 규칙에 따라 자동으로 커밋한다.

## $ARGUMENTS

커밋 메시지 힌트 (선택적). 미지정 시 변경사항 분석으로 자동 생성.

---

## Phase 1: 변경사항 확인

| 단계 | 행위 |
|------|------|
| 1 | `git status` 실행 — staged / unstaged / untracked 파일 목록 수집 |
| 2 | 변경사항이 없으면 안내 메시지 출력 후 중단 |
| 3 | `git diff` 및 `git diff --staged` 실행 — 변경 내용 파악 |
| 4 | 현재 브랜치명 및 최근 커밋 메시지 확인 |

## Phase 2: 커밋 메시지 생성

| 단계 | 행위 |
|------|------|
| 1 | 변경사항 분석 — type(feat/fix/docs/refactor/test/chore) + scope + subject 결정 |
| 2 | `$ARGUMENTS` 힌트가 있으면 반영 |
| 3 | Conventional Commits 형식으로 메시지 생성: `type(scope): subject` |
| 4 | 파이프라인 진행 중이면 태스크 ID 포함: `type(scope): T-{ID}-NN subject` |

> **사용자 확인 없이 즉시 Phase 3로 진행한다.**

> 파이프라인 Micro-tasking 커밋 규칙: [pipeline.md](../.claude/rules/pipeline.md) §3 참조

## Phase 3: 커밋 실행

| 단계 | 행위 |
|------|------|
| 1 | **Staging 기준**: 현재 태스크(plan.md Tasks 태스크 ID)와 직접 관련된 파일만 stage. 관련 없는 파일은 제외 — 파이프라인 외 호출 시에는 변경된 파일 전체 중 논리적으로 묶이는 파일만 stage |
| 2 | `git commit` 실행 |
| 3 | 커밋 결과 출력 (해시, 메시지, 변경 파일 수) |
