## 프레임워크 검증

프레임워크 파일의 구조적 무결성을 검증한다.

| 카테고리 | 검증 항목 | 세부 기준 |
|----------|---------|---------|
| **구조** | 필수 파일 존재 | CLAUDE.md, `.claude/rules/pipeline.md`, `.claude/rules/docs-writing.md`, `.claude/rules/coding.md`, 각 스킬의 `SKILL.md`, 각 커맨드 `.md` |
| **참조** | 경로 정합성 | CLAUDE.md `Key References` 섹션의 모든 경로가 실제 파일로 존재 |
| **규칙** | CLAUDE.md 줄 수 | CLAUDE.md `Framework: Forbidden` 기준 준수 (현재 200줄 제한) |
| **규칙** | frontmatter paths | Rules 파일의 `paths:` 필드 vs CLAUDE.md `Framework: Rules Delegation` 테이블 일치 |
| **일관성** | MCP 도구명 | 모든 스킬의 `### Linear MCP` 섹션에서 동일 도구명 사용 (`save_issue`, `get_issue`, `save_comment`, `list_issues`, `list_teams`) |
| **일관성** | 커맨드→스킬 매핑 | 각 커맨드가 참조하는 스킬 경로가 실제 존재 |
| **일관성** | 게이트 라벨 | 스킬 Process 테이블의 (G1)~(G4) 라벨이 §2 게이트 정의와 일치 |

| 단계 | 행위 |
|------|------|
| 1 | CLAUDE.md 읽기 — `Key References` 섹션에서 참조 경로 목록 추출 |
| 2 | 각 참조 경로의 실제 파일 존재 확인 |
| 3 | Rules 파일의 paths frontmatter vs CLAUDE.md `Framework: Rules Delegation` 테이블 비교 |
| 4 | CLAUDE.md 줄 수 확인 |
| 5 | 스킬별 Linear MCP 도구명 일관성 확인 |
| 6 | 검증 결과 테이블 출력 (PASS/FAIL + 근거) |
